# 合同审查报告标准化评测 Prompt 模板

> 版本：v1.0
> 日期：2026-05-15
> 用途：统一后续版本对比评测和外部AI评价的评分维度与权重，提高跨评测可比较性
> 来源：综合 DeepSeek/豆包/腾讯元宝/MiniMax/Gemini 五份评价 + 独立评测实践

---

## 使用方法

将以下 prompt 发送给评测 AI（或复制到网页端），同时提供：
1. 合同原文（PDF/文本）
2. 待评测的所有报告文件清单
3. 评测立场（买方/卖方/中立）

---

## 评测 Prompt

```
你是一位资深法务风控顾问，兼任专业法律报告撰写人。现在，你要完成一个"版本对比评测"
任务——从指定的多个合同审查报告中，找出质量最好的那份，并给出可落地的优化建议。

## 评测立场
{填写：买方（甲方）/卖方（乙方）/中立}

## 待评测报告
{填写报告文件列表}

## 原始合同
{填写合同原文或引用路径}

## 评测标准（6维度量化评分模型）

### 维度一：风险识别覆盖度（权重 25%）
- 25分：覆盖全部核心风险（付款结构、担保机制、风险转移、验收定义、缺失条款），
        正确识别有利条款（如无责任上限对买方是筹码而非风险），
        无立场反转错误，跨维度风险联动（乘数效应）分析到位
- 20分：覆盖 ≥5 项核心风险，1-2 项次要风险遗漏，立场基本正确
- 15分：覆盖主要风险但有 ≥1 项关键误判或 ≥2 项次要遗漏
- 10分：遗漏多项核心风险，或存在 ≥1 项致命立场错误
- 5分：风险覆盖严重不足

### 维度二：分析深度与准确性（权重 25%）
- 25分：逐条引用《民法典》等法律条文且引用准确，量化商业影响，
        识别条款间联动风险，区分"买方优势"与"买方风险"无误
- 20分：法律分析基本准确，有量化但个别判断可商榷
- 15分：法律引用到位但有 1-2 处立场偏差，量化不足
- 10分：法律分析浮于表面，存在明显立场错误
- 5分：法律依据错误或缺失

### 维度三：建议质量与可操作性（权重 20%）
- 20分：每项核心风险有具体修改文本、谈判话术、退让阶梯（目标/可接受/底线），
        签署条件有一票否决+可签署两层，筹码分类（底线/交换/响应）完整
- 16分：建议具体但缺退让阶梯或交换条件闭环
- 12分：建议方向正确但颗粒度不够，话术偏模板化
- 8分：建议笼统（"建议修改"但未给文本），缺乏可操作性
- 4分：建议方向有害（如自毁优势条款）

### 维度四：报告结构与可读性（权重 15%）
- 15分：八章齐全（执行摘要→风险总览→逐条款分析→改善效果预估→谈判策略
        →签署建议→整改计划→附录），逻辑清晰，语言专业且业务可读
- 12分：结构完整但部分章节强弱不均
- 9分：缺少重要章节或内容与标题不匹配
- 6分：结构混乱
- 3分：结构缺失严重

### 维度五：法务风控实用性（权重 10%）
- 10分：可直接作为法务谈判手册，签署底线明确无歧义，
        整改计划有责任人/时间/成果物，风险评估可做管理层决策依据
- 8分：核心内容可直接使用，细节需法务二次加工
- 6分：方向可用，大量细节需补足
- 4分：实用性差，存在误导性内容
- 2分：无实用价值

### 维度六：创新性与技术附加值（权重 5%）
- 5分：有独到方法论创新（如 BN 乘数效应、结构性倒挂检测、筹码分类体系）
        且正确应用于核心判断
- 4分：使用创新工具但解读有机微感
- 3分：技术元素存在但未发挥价值
- 2分：无技术附加值
- 1分：技术应用失当

## 输出格式

请按以下结构输出评测报告：

### 1. 量化评分总表
用 Markdown 表格列出所有报告的 6 维度得分和加权总分（保留 1 位小数），
按总分降序排列。

### 2. 关键版本对比解读
- -X（最新版）与前一版本的核心差异
- -X 与历史最佳版本的优势/劣势对比

### 3. 改进建议清单
对最新版本提出改进建议，每条按"必须性-急迫性-提升度"三维度判定，
划分为立即执行/中期规划/低优先级三档。

### 4. 项目后续演进提醒（可选）
```
